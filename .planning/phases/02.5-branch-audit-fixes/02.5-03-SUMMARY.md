## Technical Implementation

### Props Pattern Cleanup

**Before:**
```tsx
interface HealthBarProps {
  className?: string;
  props?: CypherProgressProps; // DEAD CODE - never used
  variant?: CypherProgressProps["variant"];
  value?: number;
}

// Usage in enemy-health-display:
<HealthBar props={{ progressBg: healthBarColor }} />
```

**After:**
```tsx
interface HealthBarProps {
  className?: string;
  variant?: CypherProgressProps["variant"];
  value?: number;
  progressBg?: string; // Flat prop with default
}

// Usage in enemy-health-display:
<HealthBar progressBg={healthBarColor} />
```

### Theme-Aware Color Migration

**Before (friend-list):**
```tsx
ingame: `bg-[#ff0055] border-[#ff0055] shadow-[0_0_6px_#ff0055]`
```

**After:**
```tsx
ingame: `bg-red-500 border-red-500 shadow-[0_0_6px] shadow-red-500/50`
```

**Benefits:**
- Uses Tailwind's color palette (responds to theme context)
- Consistent opacity handling via `/50` opacity modifier
- Preserves visual glow intent while enabling theme compatibility
- No hardcoded values bypassing CSS custom properties

### XpBar Text Color Fix

**Before:**
```tsx
"text-[0.625rem] text-black",
"drop-shadow-[1px_1px_0_#fff] [text-shadow:1px_1px_0_#fff,...]"
```

**After:**
```tsx
"text-[0.625rem] text-foreground",
"drop-shadow-[1px_1px_0_currentColor] [text-shadow:1px_1px_0_currentColor,...]"
```

**Result:** LEVEL UP text now adapts to theme (invisible in light mode before, now properly contrasts).
