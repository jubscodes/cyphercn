Add high-contrast variant class and system preference support after the color variants (after the .variant-orange block added in Plan 01).

```css
/* ═══════════════════════════════════════════════════════════════════════════
   PROTHEUS HIGH CONTRAST MODE - WCAG AA Compliant
   Contrast ratio: 7:1+ for text, 4.5:1+ minimum
   ═══════════════════════════════════════════════════════════════════════════ */

/* Manual high-contrast toggle via class */
.theme-protheus.high-contrast {
  /* Light mode high contrast */
  --background: oklch(1 0 0);       /* Pure white */
  --foreground: oklch(0 0 0);       /* Pure black */
  --primary: oklch(0 0 0);
  --primary-foreground: oklch(1 0 0);
  --border: oklch(0 0 0);
  --ring: oklch(0 0 0);
  --muted: oklch(0.95 0 0);
  --muted-foreground: oklch(0.3 0 0);

  /* High contrast focus ring */
  --focus-ring-width: 3px;
  --focus-ring-offset: 2px;

  @variant dark {
    /* Dark mode high contrast */
    --background: oklch(0 0 0);       /* Pure black */
    --foreground: oklch(1 0 0);       /* Pure white */
    --primary: oklch(1 0 0);
    --primary-foreground: oklch(0 0 0);
    --border: oklch(1 0 0);
    --ring: oklch(1 0 0);
    --muted: oklch(0.15 0 0);
    --muted-foreground: oklch(0.8 0 0);
  }
}

/* System-level forced colors support */
@media (forced-colors: active) {
  .theme-protheus,
  .theme-protheus[class*="variant-"] {
    /* Let system handle colors */
    --background: Canvas;
    --foreground: CanvasText;
    --primary: Highlight;
    --primary-foreground: HighlightText;
    --border: CanvasText;
    --ring: Highlight;

    /* Disable decorative effects that might interfere */
    --phosphor-glow: transparent;
  }

  /* Ensure borders are visible in forced colors */
  .theme-protheus .border-dos,
  .theme-protheus .border-dos-double,
  .theme-protheus .dos-panel {
    border-color: CanvasText;
  }

  /* Disable pseudo-element overlays */
  .theme-protheus .crt-scanlines::before,
  .theme-protheus .crt-screen::before,
  .theme-protheus .crt-full::before,
  .theme-protheus .crt-full::after {
    display: none;
  }
}

/* High-contrast focus styles */
.theme-protheus.high-contrast *:focus-visible {
  outline: var(--focus-ring-width, 3px) solid var(--ring);
  outline-offset: var(--focus-ring-offset, 2px);
}

/* Prefers-contrast support (newer, less supported) */
@media (prefers-contrast: more) {
  .theme-protheus:not(.high-contrast) {
    /* Boost contrast when system requests it but user hasn't toggled manually */
    --phosphor: oklch(0.95 0.25 142);
    --phosphor-dim: oklch(0.6 0.18 142);

    /* Increase border visibility */
    --border: oklch(0.9 0.25 142);

    @variant dark {
      --foreground: oklch(0.95 0.28 142);
      --border: oklch(0.95 0.28 142);
    }
  }
}
```

Key accessibility decisions:
- High contrast mode uses pure black/white for maximum contrast (21:1 ratio)
- Focus rings are thicker (3px) and offset (2px) for visibility
- System forced-colors uses CSS system colors (Canvas, CanvasText, Highlight)
- Decorative effects (scanlines, vignette, glow) disabled in forced-colors mode
- prefers-contrast: more gives moderate boost without full high-contrast

WCAG AA requirements:
- Normal text: 4.5:1 contrast ratio (our high-contrast: 21:1)
- Large text: 3:1 contrast ratio (our high-contrast: 21:1)
- UI components: 3:1 against adjacent colors

```css
/* ─────────────────────────────────────────────────────────────────────────────
   Focus States (Accessibility)
   WCAG 2.1 requires visible focus indicators
   ───────────────────────────────────────────────────────────────────────────── */

/* Default focus for protheus elements */
.protheus *:focus-visible {
  outline: 2px solid var(--ring, currentColor);
  outline-offset: 2px;
}

/* Remove default outline when custom focus is applied */
.protheus *:focus:not(:focus-visible) {
  outline: none;
}

/* Terminal-style focus: glow effect */
.focus-glow:focus-visible {
  outline: none;
  box-shadow:
    0 0 0 2px var(--background),
    0 0 0 4px var(--primary),
    0 0 8px var(--phosphor-glow, var(--primary));
}

/* High-visibility focus for keyboard navigation */
.focus-ring-thick:focus-visible {
  outline: 3px solid var(--ring, currentColor);
  outline-offset: 3px;
}

/* Inset focus for filled elements */
.focus-inset:focus-visible {
  outline: none;
  box-shadow: inset 0 0 0 2px var(--ring, currentColor);
}

/* Skip link for keyboard users */
.skip-link {
  position: absolute;
  top: -100%;
  left: 0;
  z-index: 100;
  padding: 0.5rem 1rem;
  background: var(--background);
  color: var(--foreground);
  border: 2px solid var(--border);
  text-decoration: underline;
}

.skip-link:focus {
  top: 0;
}

/* Focus trap indicator (for modals/dialogs) */
.focus-trap-active {
  outline: 2px dashed var(--ring);
  outline-offset: 4px;
}
```

Design rationale:
- `:focus-visible` only shows on keyboard navigation, not mouse clicks
- 2px minimum outline for visibility
- Offset prevents outline from overlapping content
- Glow focus option matches terminal aesthetic
- Skip link pattern for keyboard accessibility
